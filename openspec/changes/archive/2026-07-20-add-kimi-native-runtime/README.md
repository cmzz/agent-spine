# add-kimi-native-runtime

为 agent-spine 增加 Kimi Code 原生运行载体（add-kimi-native-runtime）：镜像 add-codex-native-runtime 的四层结构——.kimi-plugin/plugin.json 第三份 manifest + manifest 内 hooks 声明（复用 provider-neutral hook 脚本，${KIMI_PLUGIN_ROOT}）；三个 kimi host-adapter skills（读 canonical commands，Kimi 有同款 Agent(subagent_type)/AskUserQuestion/todo 工具，映射近恒等 + npc init --runtime-host kimi）；npc 扩展 runtime_host=kimi（SUPPORTED_CODER_BACKENDS 加 kimi、未配置默认 in-session 复用 default_override、kimi 生成强制 Claude review + 同源拒绝、spec_writer_host_mismatch 泛化为 runtime_host 与显式 writer 不符即拒）；测试与文档。关键约束：SubagentStop 在 Kimi 不可块（仅观察），RESULT 门依赖 npc record 权威校验（沿用 D5 设计，需写入 tradeoff）；hook payload 是否含 transcript_path、自定义 subagent_type profile 能否由 plugin 提供为待验证项，设计须给出两种情况的降级路径（无 transcript_path 则 session 探测走 hook cache/不可用降级；无自定义 profile 则用默认 coder sub-agent + prompt 文件契约）。Kimi 0.27.0 无 shell 级 plugin 安装命令，本 change 不含 qlj hub 改造（另一仓库）。
